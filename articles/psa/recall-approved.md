## <a name="impact-of-a-recall-request"></a><span data-ttu-id="60d7d-137">Kesan daripada permintaan tarik balik</span><span class="sxs-lookup"><span data-stu-id="60d7d-137">Impact of a recall request</span></span>

### <a name="operational-impact"></a><span data-ttu-id="60d7d-139">Kesan operasi</span><span class="sxs-lookup"><span data-stu-id="60d7d-139">Operational impact</span></span>

<span data-ttu-id="60d7d-143">Jika permintaan tarik balik ditolak, status entri masih **Diluluskan** dan entri tersebut tidak boleh diedit oleh ahli pasukan projek atau pelulus bagi projek tersebut.</span><span class="sxs-lookup"><span data-stu-id="60d7d-143">If a recall request is rejected, the status of the entry remains **Approved**, and the entry can't be edited by the project team member or the approver for the project.</span></span>

- <span data-ttu-id="60d7d-147">Medan **Status Pengebilan** dikemas kini kepada **Dibatalkan**.</span><span class="sxs-lookup"><span data-stu-id="60d7d-147">The **Billing Status** field is updated to **Canceled**.</span></span>

<span data-ttu-id="60d7d-148">Seterusnya, entri balikan dicipta dalam jadual Aktual.</span><span class="sxs-lookup"><span data-stu-id="60d7d-148">Next, reversal entries are created in the Actuals table.</span></span> <span data-ttu-id="60d7d-149">Untuk mencipta entri balikan, sistem menyalin nilai medan daripada aktual sebenar.</span><span class="sxs-lookup"><span data-stu-id="60d7d-149">To create reversal entries, the system copies over the field values from the original actuals.</span></span> <span data-ttu-id="60d7d-150">Satu-satunya nilai yang tidak disalin adalah nilai kuantiti.</span><span class="sxs-lookup"><span data-stu-id="60d7d-150">The only values that aren't copied over are the quantity values.</span></span> <span data-ttu-id="60d7d-151">Sebaliknya, nilai ini dibalikkan.</span><span class="sxs-lookup"><span data-stu-id="60d7d-151">These values are reversed instead.</span></span> <span data-ttu-id="60d7d-152">Aktual balikan dicipta untuk **Kos** dan aktual **Jualan Tidak Dibilkan**.</span><span class="sxs-lookup"><span data-stu-id="60d7d-152">Reversed actuals are created for both **Cost** and **Unbilled Sales** actuals.</span></span> <span data-ttu-id="60d7d-153">Medan **Status Pelarasan** pada aktual yang diterbalikkan ditetapkan kepada **Tidak boleh diselaraskan** dan medan **Status pengebilan** ditetapkan kepada **Dibatalkan**.</span><span class="sxs-lookup"><span data-stu-id="60d7d-153">The **Adjustment Status** field on the reversed actuals is set to **Unadjustable**, and the **Billing status** field is set to **Canceled**.</span></span> <span data-ttu-id="60d7d-154">Disebabkan perubahan ini, perbelanjaan yang direkodkan dan tunggakan hasil pada projek tidak lagi menjelaskan jumlah yang diwakili oleh aktual ini.</span><span class="sxs-lookup"><span data-stu-id="60d7d-154">Because of these changes, the recorded spending and the revenue backlog on the project will no longer account for the amounts that these actuals represent.</span></span>
